# 🎁 popupinfo

全国の **コスメ・日用品の「無料サンプルがもらえる」イベント／ポップアップ** を
毎日自動で収集して、**現在地から近い順・エリア・キーワード**で探せる Web アプリです。

## 主な機能

- 🔍 ブランド・会場・エリア・キーワードで検索
- 📍 「近い順」表示（位置情報は端末内だけで使い、保存しません）
- 🗾 エリア（都道府県）で絞り込み／位置情報オフ時は既定順（東京→神奈川→大阪…）
- 🆕 「今週の新着」を強調
- ♥ お気に入り（端末内に保存。ログイン不要）
- 🎁 「無料サンプル」明確なものを優先表示、曖昧なものは「要確認」表示
- 🤖 PR TIMES から**毎日自動収集**（複数キーワード・差分更新）

## 設計のポイント（決定事項）

| 項目 | 決定 |
| --- | --- |
| 1件の単位 | イベント／ポップアップ（会場＋期間、終了で消える） |
| ジャンル | コスメ＋日用品に限定 |
| エリア | 全国（都道府県判定＋代表座標で「近い順」） |
| 重複判定 | 会場＋開催期間が同じならほぼ同じ |
| 更新方式 | 差分更新（足す・更新・終了分だけ消す。初掲載日 firstSeen を保持） |
| 信頼性 | 無料サンプルが明確なものを優先、曖昧は「要確認」 |
| 計測 | まず表示回数のみ（Cookieなしのアクセス解析） |
| 位置情報 | 端末内のみで使用・保存しない |
| 速度 | 一覧は「もっと見る」で区切って表示 |
| 再訪導線 | 端末内お気に入り＋今週の新着 |
| コスト | 完全無料の範囲（外部の有料APIは未使用） |

## セットアップ（ローカル）

Node.js 18 以上が必要です。

```bash
npm install
npm run dev      # http://localhost:3000
```

## データ収集

```bash
npm run collect  # data/events.json を更新（終了分は自動削除）
```

毎日自動で回す設定は `.github/workflows/daily-collect.yml`（GitHub Actions）。
収集が0件の日は異常としてジョブが失敗し、GitHub から通知が届きます（収集元の故障検知）。

## 技術スタック

- Next.js 14 (App Router) + React 18 + TypeScript
- 自作CSS
- 収集: Node.js + cheerio（`scripts/collect.mjs`、アダプタ方式）
- アクセス解析: @vercel/analytics（表示回数）

## 注意

掲載情報は自動収集・要約のため、実際と異なる場合があります。お出かけ前に必ず公式情報をご確認ください。
